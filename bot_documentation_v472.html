1. Vue d'ensemble & Architecture

Le bot ecoute jusqu'a 9 canaux Telegram en parallele via Telethon. Des qu'un message est recu :

  1. Filtre le spam et les messages non-trading
  2. Parse le signal (direction, entry, zone, TPs, SL)
  3. Valide (spread, SL, conflit, horaire, news, max positions, P&L quotidien)
  4. Execute l'ordre sur MetaTrader 5 (market ou limit)
  5. Gere le BE (Break Even) et le trailing SL
  6. Limite le P&L quotidien
  7. Log les trades dans Supabase
Telegram Channels (1-9) ↓ Signal Parser V7.4.14 → TradeSignal {direction, entry, zone, TPs, SL} ↓ Validation : spread / SL / conflit / horaire / news / max positions / P&L quotidien ↓ MT5Bridge → place_market_order / place_limit_order ↓ TradeManager → BE + Trailing SL + Grade + P&L quotidien (poll 1s) ↓ Supabase Logger → trades / events / TP hit / SL hit

2. Signal Parser V7.4.14

2.1 Pipeline de normalisation

15 etapes de nettoyage avant extraction :

Texte brut : " 🟢 BUY X AU/USD @3 3 4 0 TP¹ 3345 SL: 3330 " ↓ 1-2. Supprimer caracteres invisibles (zero-width, U+00A0) 3. Convertir superscripts : ¹→1, ²→2 4-5. Supprimer astérisques, S/L → SL 6-7. Supprimer () [] {} emojis guillemets, separators → espace 8-9. Fusionner symboles : X AUUSD → XAUUSD, XXAU/USD → XAUUSD 10. Supprimer prefixes parasites 11-15. Normaliser espaces + UPPER ↓ Resultat : "BUY XAUUSD 3340 TP1 3345 SL 3330"

2.2 Extraction

ComposantMethodeVariantes
DirectionRegex + mappingBUY, SELL, LONG, SHORT, SEL, PURCHASE, ⬆/⬇
SymboleRegex + SYMBOL_MAPGOLD, XAU/USD, XXAUUSD, XAGUSD, USOIL, BTCUSD
Zone8 patterns4199/4203, 4199:4203, ENTRY 3240, BUY 4512
TPsRegex TP/T/TAKE PROFITTP1, TP¹, TARGET, TGT
SLRegex SL/STOPSL, STOP LOSS, SL BREAKOUT

2.3 Generation automatique

SL auto : Si TPs mais pas de SL → distance moyenne TPs × 1.5
TP auto : Si SL mais pas de TP → RR_RATIO_DEFAULT=1.5
Quick Alert : Si ni TP ni SL → SL provisoire ±10 points, confiance 0.2

2.4 Filtre anti-spam

Mots exclus : hit, pips, closed at, good morning, subscribe, vip...

Exception : si le message contient BUY/SELL, il n'est PAS filtre.

2.5 Tri des TPs

// BUY : TPs croissants : [3340, 3345, 3350] // SELL : TPs decroissants : [3350, 3345, 3340]

4. Alerte rapide + Merge

4.1 Alerte rapide

GOLD SELL NOW 4199

Le bot detecte une alerte rapide (direction + entry, pas de TPs/SL). Il place un ordre avec SL provisoire (10 points) et TP par defaut (RR=1.0).

Quick Alert : GOLD SELL NOW 4199 ↓ Prix entre SL et entry ? ├─ OUI → MARKET SL=4209, TP=4189 └─ NON → LIMIT @ 4199, SL=4209, TP=4189

4.2 Signal complet → Merge

GOLD SELL NOW 4199 LIMIT 4203 TP1 ▶️ 4195 | TP2 ▶️ 4191 | TP3 ▶️ 4187 ❌SL --------> 4209
Signal complet recu : ↓ Verifier quick alert existante : ↓ Scenario 1 : Quick alert REMPLI (position ouverte) ├─ Modifier SL : provisoire → reel (4209) ├─ Poser TP = 4187 (tp_final) └─ Poser LIMIT SELL @ 4203 (merge) Scenario 2 : Quick alert PAS ENCORE REMPLI (pending) ├─ Modifier SL + TP sur l'ordre pending └─ Poser LIMIT SELL @ 4203 (merge) Cas special : SL/TP touche AVANT le signal complet └─ IGNORER le signal complet
Config : QUICK_ALERT_SL_OFFSET=10.0, DEFAULT_ALERT_TP_RR=1.0

5. Cas d'execution des ordres

5.1 Prix unique (is_single_price)

ScenarioCondition (BUY)Action
S1SL < prix < entryMARKET — meilleur prix, RR ameliore
S2entry < prix < TP1LIMIT @ entry — attendre retour
S3sinonANNULE

5.2 Signal avec zone — CAS 1 (dans la zone)

CAS 1 : zone_low ≤ prix ≤ zone_high ├─ 50% lot → MARKET @ prix actuel └─ 50% lot → LIMIT entre SL et zone

5.3 Signal avec zone — CAS 2 (hors zone)

CAS 2 : prix hors zone ↓ CAS 2-a : zone < prix < TP1 ├─ 50% → MARKET @ prix actuel └─ 50% → LIMIT @ l'autre bout de la zone CAS 2-b : TP1 ≤ prix ≤ TP2 ├─ LIMIT_1 @ zone_edge └─ LIMIT_2 @ zone_opposite Hors zone : prix > TP2 (BUY) ou prix < TP2 (SELL) └─ ANNULE

5.4 TP_TRIGGER ajustement

Si le signal n'a pas assez de TPs, le TP_TRIGGER est ajuste au dernier TP disponible (au lieu d'ignorer le signal).

6. Gestion BE + Trailing

6.1 Declenchement

P&L-based trigger : Le BE se declenche quand le P&L minimum des positions actives atteint PNL_TRIGGER_USD (8$).
SituationTriggerMethode
MARKET rempli (position ouverte)P&Lpos.profit ≥ 8$
LIMIT seulement (pas de position)TP_TRIGGEROHLC M1 : prix a touche TP3 ?
MARKET + LIMIT (CAS 1)P&Lpos.profit ≥ 8$

6.2 Trailing SL

ParametreValeurRole
TRAIL_RATIO_R12$Le SL bouge de R1$
TRAIL_RATIO_R23$Le prix doit bouger de R2$
Exemple SELL @ 3340 : Prix = 3332, P&L = 8$ → BE active SL = 3340 Prix = 3329 (bouge 3$) → SL = 3340 - 2 = 3338 Prix = 3326 (bouge 3$) → SL = 3338 - 2 = 3336 Prix = 3323 (bouge 3$) → SL = 3336 - 2 = 3334 ...jusqu'au TP ou retournement

6.3 CAS 1 au declenchement

Sous-casConditionAction
2-aLIMIT non remplieAnnuler LIMIT, BE + trailing sur MARKET
2-bLIMIT remplieFermer MARKET, BE + trailing sur LIMIT

6.4 CAS 2-b au declenchement

Sous-casConditionAction
3-b-1Aucun limit rempliAnnuler les 2 limits
3-b-2L1 rempli seulementBE sur L1, annuler L2
3-b-3Les 2 remplisFermer L1, BE + trailing sur L2

6.5 Quick Alert BE

Meme logique : P&L trigger → BE sur la position restante (MARKET ou LIMIT).

7. P&L Quotidien

Chaque seconde : total = P&L realise + P&L flottant ↓ total ≥ DAILY_PROFIT_LIMIT (30$) ? ├─ OUI → Fermer TOUT + annuler TOUS les ordres └─ NON → Continuer Reset : A minuit UTC, compteur remis a zero
ParametreValeurDescription
DAILY_PROFIT_LIMIT30.0Seuil de profit quotidien en $ (0 = illimite)

8. Gestion des conflits

Si un signal BUY arrive alors qu'une position SELL existe sur le meme symbole :

  1. Annuler tous les ordres pending du conflit
  2. Fermer les positions ouvertes
  3. Executer le nouveau signal
Note : Desactive en mode DEMO.
