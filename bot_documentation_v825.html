1. Vue d'ensemble & Architecture

Le bot écoute jusqu'à 9 canaux Telegram en parallèle via Telethon. Dès qu'un message est reçu :

  1. Filtre le spam et les messages non-trading
  2. Parse le signal (direction, entry, zone, TPs, SL)
  3. Valide (spread, SL, conflit, horaire, news, max positions, P&L quotidien)
  4. Exécute l'ordre sur MetaTrader 5 (market ou limit)
  5. Gère le BE (Break Even) et le TP_FIXED (fermeture manuelle)
  6. Limite le P&L quotidien
Telegram Channels (1-9) ↓ Signal Parser V7.4.14 → TradeSignal {direction, entry, zone, TPs, SL} ↓ Validation : spread / SL / conflit / horaire / news / max positions / P&L quotidien ↓ MT5Bridge → place_market_order / place_limit_order ↓ TradeManager → BE + TP_FIXED + P&L quotidien (poll 1s) ↓ Alertes Telegram → notifications de chaque événement

2. Signal Parser V7.4.14

2.1 Pipeline de normalisation

15 étapes de nettoyage avant extraction :

Texte brut : " 🟢 BUY X AU/USD @3 3 4 0 TP¹ 3345 SL: 3330 " ↓ 1-2. Supprimer caractères invisibles (zero-width, U+00A0) 3. Convertir superscripts : ¹→1, ²→2 4-5. Supprimer astérisques, S/L → SL 6-7. Supprimer () [] {} emojis guillemets, séparateurs → espace 8-9. Fusionner symboles : X AUUSD → XAUUSD, XXAU/USD → XAUUSD 10. Supprimer préfixes parasites 11-15. Normaliser espaces + UPPER ↓ Résultat : "BUY XAUUSD 3340 TP1 3345 SL 3330"

2.2 Extraction

ComposantMéthodeVariantes
DirectionRegex + mappingBUY, SELL, LONG, SHORT, SEL, PURCHASE, ⬆/⬇
SymboleRegex + SYMBOL_MAPGOLD, XAU/USD, XXAUUSD, XAGUSD, USOIL, BTCUSD
Zone12 patterns4199/4203, 4199:4203, ENTRY 3240, BUY 4512
TPsRegex TP/T/TAKE PROFITTP1, TP¹, TARGET, TGT
SLRegex SL/STOPSL, STOP LOSS

2.3 Génération automatique

SL auto : Si TPs mais pas de SL → distance moyenne TPs × 1.5
TP auto : Si SL mais pas de TP → RR_RATIO_DEFAULT=1.5
Quick Alert : Si ni TP ni SL → SL provisoire ±10 points, confiance 0.2

2.4 Filtre anti-spam

Mots exclus : hit, pips, closed at, good morning, subscribe, vip...

Exception : si le message contient BUY/SELL, il n'est PAS filtré.

2.5 Tri des TPs

// BUY : TPs croissants : [3340, 3345, 3350] // SELL : TPs décroissants : [3350, 3345, 3340]

CAS 1 : zone_low ≤ prix ≤ zone_high ├─ 50% lot → MARKET @ prix actuel └─ 50% lot → LIMIT entre SL et zone

4.3 Signal avec zone — CAS 2-a (prix entre TP1 et zone)

CAS 2-a : tp1 < prix < zone_low (SELL) ou zone_high < prix < tp1 (BUY) ├─ 50% → MARKET @ prix actuel └─ 50% → LIMIT @ l'autre bout de la zone

4.4 Signal avec zone — CAS 2-b (prix entre TP1 et TP2)

CAS 2-b : tp2 < prix < tp1 (SELL) ou tp1 < prix < tp2 (BUY) ├─ LIMIT_1 @ zone_edge └─ LIMIT_2 @ zone_opposite

5. Gestion BE + TP_FIXED

5.1 Fonctionnement général

Signal exécuté → positions ouvertes ↓ Chaque seconde (_check_all) : ↓ PNL d'une position ≥ PNL_TRIGGER_USD (8 pts) ? ├─ OUI → BE ACTIVé │ ├─ Annuler les ordres pending │ ├─ 1 position : SL @ entry_price │ ├─ 2 positions : SL au médian │ ├─ TP = TP final du signal (jamais modifié) │ └─ target_gain = TP_FIXED_GAIN_USD × N └─ NON → continuer ↓ total_pnl ≥ target_gain ? ├─ OUI → TP_FIXED → fermer manuellement toutes les positions └─ NON → continuer (broker TP peut fermer en premier)

6.2 Fusion (QA + signal complet)

Signal complet reçu : ↓ Vérifier quick alert existante : ↓ Scénario 1 : Quick alert REMPLI (position ouverte) ├─ Modifier SL : provisoire → réel ├─ Poser TP = tp_final └─ Poser LIMIT @ zone_edge (merge) Scénario 2 : Quick alert PAS ENCORE REMPLI (pending) ├─ Modifier SL + TP sur l'ordre pending └─ Poser LIMIT @ zone_edge (merge) Cas spécial : SL/TP touché AVANT le signal complet └─ IGNORER le signal complet

7. TP_TRIGGER

7.1 Fonctionnement

Le TP_TRIGGER est le 3ème TP du signal (ou le 2ème, ou le 1er si moins de 3 TPs). Quand le prix atteint ce niveau et qu'aucune position n'est ouverte, les ordres pending sont annulés.

Chaque seconde (quand pas de positions ouvertes) : ↓ tp_trigger = tps[2] (3ème TP) ↓ Prix a atteint tp_trigger ? ├─ OUI → Annuler TOUS les ordres pending └─ NON → continuer

7.2 Exemple

Signal : SELL zone 4075-4080, TP1=4072, TP2=4069, TP3=4066 tp_trigger = tps[2] = 4066 Limit_1 @4075 rempli, Limit_2 @4080 pending Prix descend à 4066 → TP_TRIGGER atteint Résultat : ===== | CH2-C2 | TP_TRIGGER | ===== SELL XAUUSD | @4080.0 | 1 ordres annulés
